## Session 4 — Complete

**Decision: Approach A (no runtime validation).**
The Node CI validator (`npm run validate:data`) is the primary safety net. Runtime validation in the browser (Approach B: zod via UMD script tag) was evaluated and skipped for the initial implementation. Loader boundary locations are documented in `globals.d.ts` header comment if Approach B is ever needed.

**Deliverables:**
- `globals.d.ts` rewritten: data-shape types now derived via `z.infer<typeof Schema>` from the schema files. Zod is the single source of truth for JSON-facing shapes. Inline type definitions eliminated.
- `Program` kept as inline type (runtime-generated by `EPGDataModel`, not in any JSON schema file)
- All 57 tests pass. `npm run typecheck` clean. `npm run validate:data` all green.
